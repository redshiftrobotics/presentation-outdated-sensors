Sensors

Written and presented by AJ

What the heck? Sensors?

Sensors are things that we put on the robot

We use sensors to get information about the environment that the robot is in

Most of our sensors are standard LEGO sensors, although some are from HiTechnic

Sensors are connected to the NXT using I2C cables

(You don't need to worry about this last bit unless you're a programmer.)

The Touch Sensor

The touch sensor basically returns true or false:

Is the button pressed, or is it not pressed?

Typically we use touch sensors for limit switches

These help us know when to stop a motor so that it won't damage the robot.

The Color Sensor

The color sensor returns what color is in front of the sensor. It works at very close ranges.

Typically we use color sensors to write line following programs

This year we may use color sensors to detect when we're on the ramp.

The Light Sensor

The light sensor senses light intensity.

We tend not to use this sensor a lot, although this year we may use it for the ramp.

The Sound Sensor

The sound sensor returns the sound intensity of the environment.

We almost never use this sensor.

The Ultrasonic Sensor

The ultrasonic sensor tells us how far away something is.

It works in the same way bats do echolocation.

Typically we don't use this sensor, especially because it's very inaccurate. (We might use it for the ramp, though. Maybe.)

The HiTechnic Sensor

Psych! This is like 5 different sensors.

HiTechnic sensors all look the same, so they're labeled on the bottom.

The HiTechnic Color Sensor

The HiTechnic color sensor is exactly the same as the NXT color sensor

It's just a lot more accurate.

I said earlier that we use "color sensors". This does not include the NXT color sensor, because this one is by far superior.

The HiTechnic Infrared Seeker

The HiTechnic infrared seeker returns the intensity of infrared. We can get 5 values out of this, in a fan, from the sensor.

The HiTechnic Infrared Seeker

The HiTechnic infrared seeker returns the intensity of infrared. We can get 5 values out of this, in a fan, from the sensor.

We use this to seek the IR beacon in autonomous.

The HiTechnic Magnetic Sensor

The HiTechnic magnetic sensor detects magnetic fields.

We never use it, because there's no use for it, along with the fact that the motors severely interfere with it.

The HiTechnic Accelerometor

The HiTechnic accelerometer detects acceleration in space. It's accurate in the long term, but not in the short term.

We want to use it to help determine the robot's exact location in space during autonomous.

The HiTechnic Gyroscope

The HiTechnic gyroscope detects the robot's orientation in space. It's accurate in the short term, but not the long term.

For this reason, we want to pair the gyro and the accelerometer together to achieve long-term, exact positioning data.

This is currently an open research question.

Bonus points! The I2C multiplexor

The I2C multiplexor increases the amount of I2C ports that can be used, and hence, the amount of sensors that can be used.

Basically, it's like a USB hub, but for NXT ports.

We don't like to use it, mostly because it's too complicated to work with.
